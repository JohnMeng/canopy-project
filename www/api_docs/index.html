Canopy Documentation

Canopy is an open source project to simplify the development of Internet of Things products.

Canopy consists of embedded, server-side, and frontend components that work together to bring smart objects onto the internet.

Reference

Smart Device Description Language (SDDL) Specification

JSON-based file format that describes a device's capabilities. SDDL is used by Canopy to auto-generate code and REST endpoints.

Canopy Embedded Library API (libcanopy)

Embedded software library that communicates with the Canopy Cloud Service. It provides a convenient event loop, with a callback mechanism, to your embedded application.

Canopy Embedded Code Generation Tool (canogen)

An alternative to using the Canopy Embedded Library directly. The canogen tool will auto-generate template code based on your SDDL file.

Canopy REST API Documentation

REST API used by frontend web and mobile application developers to interact with Canopy-enabled devices.

Internals

Typically there is no need to understand the inner-workings of Canopy. That said, if you are contributing to the Canopy platform itself, or heavily customizing it, you may find this material valuable.

Canopy Websocket Protocol (CWP)

Protocol used for communication between Canopy-enabled devices and the Canopy Cloud Service.
