Introduction

Canopy is an open source Internet of Things platform.

Canopy takes a description of a physical object's capabilities, as provided in an SDDL file (more on that later), and from that description exposes a RESTful web service for monitoring and controlling the object.

Canopy's server-side components are designed to scale to, potentially, millions of connected devices. The generality and flexibility of Canopy makes it a powerful platform choice whenever cloud-management is needed.

Hardware Requirements

Canopy requires a hardware platform with similar specifications to a Raspberry Pi or Beagle Bone. Canopy does not work on the Arduino platform, at present.

Furthermore, a network connection (TCP/IP) is required, over ethernet, wifi, or 4G LTE, between the device and the Canopy Cloud Service. Canopy is designed for WiFi networks, and at present does not support Zigbee, ZWave, bluetooth, or other communication protocols.

Canopy requires a linux-based OS on the device, such as Raspian or Archlinux. Canopy has not been tested on Android or RiscOS.

Recommended System Requirements:

  • ARM SOC w/ 1GHz cpu or higher
  • 128 MB RAM
  • Ethernet, WiFi, or 4G LTE connectivity
  • Linux-based OS

Features

Canopy has several features that simplify the development of high-quality IoT products.

Automatic embedded code generation

Based on your product's SDDL file, the canogen tool will automatically generate C template code for your device. Just implement a few device-specific routines (such as writing to GPIO pins) and you've got yourself a Canopy-enabled smart product!

Sensor data collection and archiving

The Canopy Cloud Service receives periodic status updates from each connected device. It stores sensor data, device status, and control events in a Cassandra database and makes this big data available for monitoring and analysis.

Low-latency remote control

Control instructions issued from the web are forwarded to the appropriate device in a matter of milliseconds.

User account management and device sharing

The Canopy Cloud Service has built-in user account management and device sharing systems. Users can choose who else can control and monitor their devices, and leave comments about particular devices for other users to see. These features make Canopy a veritable social network for IoT.

Flexible deployment options

The Canopy Cloud Service is a golang application that can be run almost anywhere. For convenience, we maintain a hosted deployment that you can use. But if you choose, you can start up your own deployment on EC2, in your own datacenter, on your corporate LAN, or on your laptop.

Device Management Frontend

You may choose to build your own custom frontend for your Canopy-enabled products. But until that's ready, you can use our generic device management frontend to monitor and control your devices.
