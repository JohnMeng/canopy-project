1.3 Getting Started: Canopy Basics

Components of Canopy

Canopy is a collection of software tools and libraries that simplify the development of cloud-enabled products. By "cloud-enabled product", we mean any physical device with a connection to the internet that can be monitored and/or controlled over the web. Such products may include consumer appliances, industrial equipment, and other "Internet of Things" devices.

A cloud-enabled product needs software to run in several places: on the device, in the cloud, and within client-side applications. Canopy contains sub-projects for each of these:

  • Canopy Embedded Development Kit (EDK) software runs on the device.
  • Canopy Cloud Service (CCS) software runs server-side.
  • Canopy Application Development Kit (ADK) software runs within an application. For example: on a smartphone or in a web browser.
Depending on your role, you may need to use just one of these components. A firmware developer will likely only use the Canopy EDK, whereas a frontend app developer will likely only use the Canopy ADK.

That said, we believe the best way to master Canopy is to understand all of the components and how they work together. Throughout this guide, we intermix our discussion to talk about all components. You can skip past sections if you know they are not relevant to you, and sometimes will we give recommendations letting you know that you can safely skip a section.

Canopy Features

Cloud Variables

The main feature of Canopy is called Canopy Cloud Variables. A Cloud Variable is a piece of "state" (i.e., data) that can be accessed by firmare but is stored in the cloud.

Cloud Variables are a powerful concept because of their generality and their easy-of-use. Cloud Variables can be used for:

  • Remote monitoring of sensor data
  • Remote control over the web
  • Device status reporting
  • Device configuration
We will discuss Cloud Variables in detail in Chapter 2.

Account & Device Management

Canopy has a built-in permission system with cloud-based Account and Device resources. This permission system controls who can access each Device's data securely.

Protocols: REST (HTTP) and Websockets

The Canopy Cloud Service (CCS) is the core component of Canopy. The other components, namely the EDK and ADK, interact with the CCS through web-based protocols.

At this time, two protocols are supported:

  • Canopy REST API is a RESTful Web API that uses HTTP(S) with JSON payloads.
  • Canopy Websocket Protocol is a websocket protocol with JSON payloads.
The EDK can be configured to use either protocol, whereas the ADK currently only supports the REST API.

Embedded and application developers may use the REST API directly. This allows Canopy to be used from practically any language and platform. However, if EDK/ADK support is available for your language & platform, it is easier to use those libraries.

What's Inside?

Let's take a brief look at what is inside each sub-project.

Canopy Embedded Development Kit (EDK)

The EDK contains tools and software for embedded development using Canopy.

  • libcanopy is a C library that runs on the device as part of the "firmware".
  • canotool is a multi-purpose tool that helps with device provisioning and other tasks.
The EDK also contains:
  • Other libraries that libcanopy depends on (including third-party libraries like libwebsockets).
  • Unit tests and sample programs.
  • A build system for compiling all of the code.
The Canopy EDK is an open source project consisting of several git repositories. It uses the repo tool to manage the multiple repositories.

Canopy Cloud Service

The CCS contains Canopy's server-side components.

  • canopy-cloud-service is the main server-side process, written in Go-lang.
The CCS also contains:
  • Install scripts for installing the cloud service.
  • Configuration files for configuring the cloud service.
  • init.d scripts for starting and stopping the cloud service.
The CCS uses Cassandra for data storage. So in addition to configuring/operating the CCS, you must also configure/operate a Cassandra database cluster.

If you use our free or premium hosting solutions, we take care of operating the CCS and Cassandra database for you.

The Canopy Cloud Service is an open source project available on github:
http://github.com/canopy-project/canopy-cloud.

Canopy Application Development Kit

The ADK contains libraries for application developers, providing language-specific bindings that are easier to use compared to using the REST API directly.

  • canopy.js provides a Javascript wrapper around the Canopy REST API.

Currently, only Javascript is supported, although we hope to add bindings for other languages soon.

The Canopy ADK is an open source project available on github:
http://github.com/canopy-project/canopy-js-client.
