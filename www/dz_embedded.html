Canopy Embedded Development

Making things smart with Canopy is easy. However, it does require that your device can connect to the internet and can compile and run linux-based binaries. As a rule-of-thumb, Canopy requires hardware and software comparable to what comes on a Raspberry Pi or Beaglebone.

Embedded Development Kit

The Canopy Embedded Development Kit is entirely open source. There are a few pieces that you should know about:

libcanopy
Embedded software library that interacts with the Canopy Cloud Service.

cano
Command-line utility. Running "cano gen <SDDL_FILE>" generates embedded code for your product based on your SDDL file.

/etc/canopy/canopy.conf.json
Embedded configuration file.

Smart Product Development Process

The process of developing a smart product with Canopy generally goes like this:
  1. Install the Canopy Embedded Development Kit on your device.
  2. Create an SDDL file describing your product's sensors and controls .
  3. Run "cano gen <SDDL_FILE>" to generate embedded C code for your device.
  4. Modify the generated code stubs. For example. you may need to write to GPIO pins.
  5. Compile and run. Your device will begin communicating with our cloud servers.
  6. Log in to http://canopy.link to monitor and control your device.
  7. Optionally, develop a customized frontend for your device, using our REST API or client libraries.
See the Embedded Tutorial for a walkthrough of these steps.
